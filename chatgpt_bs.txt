# Distributed Heterogeneous Scheduler — TFM

## Overview

This project explores **task scheduling strategies in heterogeneous distributed systems** using a real home-lab cluster composed of low-power ARM devices, legacy CPUs, laptops, desktop workstations and GPU-capable nodes.

The goal is to design, implement and evaluate a distributed scheduler capable of assigning workloads efficiently depending on:

* node capabilities
* workload characteristics
* current system load
* energy/performance tradeoffs

The project begins with classical scheduling policies and evolves toward adaptive and machine-learning-assisted scheduling strategies.

---

# Core Research Question

> How does hardware heterogeneity affect scheduling strategies in distributed systems, and can machine-learning-assisted scheduling outperform classical heuristics in heterogeneous edge/home-lab environments?

---

# High-Level Objectives

* Build a distributed execution framework using Docker containers
* Create a scheduler capable of assigning jobs to heterogeneous nodes
* Compare classical scheduling strategies:

  * Round Robin
  * Least Loaded
  * Weighted Scheduling
  * Capability-Aware Scheduling
* Benchmark multiple workload types:

  * CPU-bound
  * Memory-bound
  * IO-bound
  * GPU-bound
* Collect runtime metrics and energy estimations
* Implement predictive scheduling using ML models
* Evaluate tradeoffs:

  * throughput
  * latency
  * makespan
  * energy efficiency
  * resource utilization

# Main Components

## Scheduler

Central decision-making service.

Responsibilities:

* receive node heartbeats
* store node metrics
* receive jobs
* select best node
* dispatch workloads
* collect execution statistics

---

## Agents

Runs on every node.

Responsibilities:

* collect metrics
* execute jobs
* report node capabilities
* benchmark local hardware
* report execution results

---

## Jobs
